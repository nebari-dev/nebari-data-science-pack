# Reference

Look-up material for the pack — values, releases, and reader
personas. The pages here are written for skimming, not reading
cover-to-cover.

If you are looking for the operational material (architecture,
configuration, troubleshooting), it lives in
[Deployment →](../get-started/) instead. Reference is for facts you
need to confirm, not workflows you need to follow.

## What's in this section

- **[`values.yaml` reference →](./values)** — every chart value with
  its default, type, and a one-line description. Grouped by area
  (NebariApp, singleuser culler, shared storage, Nebi, RBAC
  bootstrap, JupyterHub subchart overrides).
- **[Release notes →](./release_notes)** — what shipped in each tagged
  release, with links to the GitHub releases page and the chart's
  Helm repository.
- **[Personas →](./personas)** — who this documentation is written
  for and where each persona should start.
