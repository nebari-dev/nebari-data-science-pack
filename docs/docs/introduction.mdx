# Welcome to the Nebari Data Science Pack

**The Nebari Data Science Pack is a Helm chart that deploys
[JupyterHub](https://z2jh.jupyter.org/) with
[jhub-apps](https://jhub-apps.nebari.dev/) on Kubernetes — purpose-built
for collaborative data science teams running on a Nebari cluster.**

It bundles the [Zero to JupyterHub](https://z2jh.jupyter.org/) chart,
the jhub-apps launcher for data apps, and an optional Nebari-aware
routing / TLS / OIDC layer into a single install. Bring it up once and
your team gets JupyterLab, per-group shared storage, profile-based
sizing, and (optionally) single sign-on against Keycloak — the same
day.

The right starting point depends on what you're trying to do:

  or Nebari cluster → Learn how to [deploy the pack →](./get-started/deploy)

## Why use this pack?

Standing up a production JupyterHub on Kubernetes from scratch is a
chore. You typically have to:

- Template the upstream z2jh chart with the right hub image, custom
  config files, profile list, and culler settings.
- Add jhub-apps to the hub container and wire its service port through
  the proxy NetworkPolicy.
- Build an ingress with TLS and bolt OIDC auth onto JupyterHub's
  `GenericOAuthenticator` against Keycloak.
- Wire per-group shared directories that need a `ReadWriteMany`
  StorageClass — something most cloud providers don't ship with.

The pack does all of this for you, with defaults aimed at notebook
users on a Nebari cluster. Install one chart and your team gets
JupyterLab, jhub-apps, and (optionally) a Keycloak-backed login on
day one.

## Key features

- **One-chart install** — z2jh JupyterHub, jhub-apps, hub config
  files, and the rest of the supporting Kubernetes resources come up
  from a single Helm chart.
- **Nebari-aware defaults** — when `nebariapp.enabled: true` the chart
  emits a NebariApp resource that the nebari-operator picks up to
  provision Envoy Gateway routing, TLS, and the Keycloak OIDC client.
- **jhub-apps integration** — the hub image ships with
  [jhub-apps](https://jhub-apps.nebari.dev/) pre-installed and the
  chart wires the `JAppsConfig`, service port, and proxy
  NetworkPolicy.
- **Per-group shared storage** — `/shared/<group>` directories appear
  in every user pod, backed by a `ReadWriteMany` PVC. A transitional
  in-cluster NFS server is included for clusters without a native RWX
  StorageClass.
- **Nebi integration** — when the
  [nebi-pack](https://github.com/nebari-dev/nebari-nebi-pack) is also
  installed, JupyterLab pods auto-connect to your team's shared
  environments using the user's Keycloak token.

## What this pack is not

The pack is tuned for **interactive notebook + jhub-apps workloads**
on Nebari. It is not a generic JupyterHub distribution — it pins a
Nebari hub image with jhub-apps pre-installed, ships
KeyCloakOAuthenticator wiring, and assumes the nebari-operator manages
routing in production. For model serving (Ray Serve / HTTP endpoints)
use the [Ray Serve pack](https://github.com/nebari-dev/nebari-rayserve-pack)
instead.

## At a glance

| Aspect | Details |
|---|---|
| Upstream JupyterHub chart | [Zero to JupyterHub](https://z2jh.jupyter.org/) — exact version pinned in [`Chart.yaml`](https://github.com/nebari-dev/nebari-data-science-pack/blob/main/Chart.yaml) |
| jhub-apps | pre-installed in the Nebari hub image |
| Default authenticator | `dummy` (any username/password — for local dev) |
| Production authenticator | `GenericOAuthenticator` against Keycloak |
| Default access pattern | port-forward (standalone), Envoy Gateway HTTPS (Nebari) |
| Shared storage | per-group `/shared/<group>` on a `ReadWriteMany` PVC |
| Maturity | experimental |
| Source | [github.com/nebari-dev/nebari-data-science-pack](https://github.com/nebari-dev/nebari-data-science-pack) |

:::note[Defaults reflect chart v0.1.0-alpha.13]

For the exact pinned versions of every dependency and image tag, see
[`Chart.yaml`](https://github.com/nebari-dev/nebari-data-science-pack/blob/main/Chart.yaml)
and [`values.yaml`](https://github.com/nebari-dev/nebari-data-science-pack/blob/main/values.yaml)
in the repo.

:::
